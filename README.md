# 出差管理系统

作品集项目一（流程 / 审批类）。员工提交出差申请，主管审批，状态自动流转。

## 技术栈

| 层 | 用什么 | 干什么 |
|---|---|---|
| 后端 | Python + FastAPI | 处理业务逻辑、对外提供“接口” |
| 数据库 | SQLite（travel.db 一个文件） | 存用户和出差申请 |
| 前端 | （W3 再加） | 给人看的界面 |

## 怎么跑起来

```bash
# 1. 进入 backend 目录
cd backend

# 2. 用项目自带的虚拟环境启动后端服务
../.venv/bin/uvicorn main:app --reload

# 3. 浏览器打开：
#    http://127.0.0.1:8000/ui/     ← 网页界面（登录、提交、审批）
#    http://127.0.0.1:8000/docs    ← 接口操作台（给开发/对接口用）
```

不想启服务、只想快速验证后端通不通：

```bash
cd backend
../.venv/bin/python smoke_test.py
```

## 部署上线（拿到可分享的链接）

本项目已备好部署配置（`requirements.txt` / `render.yaml` / `Procfile`）。推荐用 Render 免费部署：

1. 把代码推到 GitHub（一个仓库）
2. 登录 render.com → New → Blueprint → 选这个仓库 → 自动按 `render.yaml` 部署
3. 部署完成后会得到一个公网地址，访问 `https://<你的应用>.onrender.com/ui/` 即可

> 注：免费实例会在闲置后休眠，首次打开需等十几秒唤醒；SQLite 数据在重启后会重置（演示足够）。
